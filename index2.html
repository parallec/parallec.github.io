Send Aggregated API Data Anywhere

Generic response hander/conext enables you send aggregated data to Kafka, Zabbix, Elastic Search, Redis, MongoDb. Check example here to aggregate 1000 website information and send its status code to elastic search and visualized in Kibana in 30 lines of code.


import io.parallec.core.*;
import java.util.Map;

ParallelClient pc= new ParallelClient();
pc.prepareGet("/validateInternals.html")
    .setMaxConcurrency(1000)
    .setTargetHostsFromString("parallec.github.io www.jeffpei.com www.restcommander.com")
    .execute(new ParallecResponseHandler() {
        @Override
        public void onCompleted(ResponseOnSingleTask res,
                Map responseContext) {
            String cpu = new FilterRegex(".*CPU-Usage-Percent\\s*(.*?).*")
            .filter(res.getResponseContent());
            System.out.println("cpu:" + cpu + " host: " + res.getHost() );
        }
    });
pc.releaseExternalResources();

Why Parallec ?

Compared with java thread pool based solution, parallec give you worry free concurrency control without constraints on thread size. Compared with single-thread NodeJs solutions, Parallec make it possible for parallel computation-intensive response handling.
